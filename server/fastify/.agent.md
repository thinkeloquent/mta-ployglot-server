# mta-fastify-server (mjs)

Runtime app. A standalone Fastify server that boots on top of `fastify-server` (the TS half of the `mta-ployglot-server-bootstrap` sibling, symlinked into `ployglots/`), with auto-discovered environment / lifecycle / route addons.

## Run

```bash
# from repo root
make dev-fastify     # foreground, --watch
# or
make dev             # background both runtimes, logs under .dev/
```

Inside this package:

```bash
cd server/fastify
npm install
node main.mjs        # uses env PORT (default 51000), HOST, APP_ENV, LOG_LEVEL
```

This package is `private: true`. Do not `npm install` it from downstream.

## Extension Contract

The bootstrap's `routeAddon`, `lifecycleAddon`, and `environmentAddon` auto-discover files under `config/` by filename suffix. Drop a file with the matching suffix and it is loaded at boot. Numeric prefix sorts loading order.

| Directory | Suffix | Export shape | Discovery order |
|---|---|---|---|
| `config/environment/` | `.env.mjs` | side-effect module (mutates `process.env`) | ascending by numeric prefix |
| `config/lifecycles/` | `.lifecycle.mjs` | `export { onInit?, onStartup?, onShutdown? }` (or default-object with same keys) | ascending by numeric prefix |
| `config/routes/` | `.routes.mjs` | `export default async (fastify, config) => void` (or `mount`) | ascending by numeric prefix |

Adding a capability to this server means adding a file in one of those three directories â€” no edits to `main.mjs` or the bootstrap sibling required.

## Canonical Usage

A new route file:

```mjs
// config/routes/20_hello.routes.mjs
export default async function helloRoutes(fastify, config) {
  fastify.get("/hello", async () => ({
    service: config.title,
    profile: config.profile,
    hello: "world",
  }));
}
```

A new lifecycle hook:

```mjs
// config/lifecycles/15_db_pool.lifecycle.mjs
export async function onInit(server) {
  server.decorate("dbPool", await createPool(process.env.DB_URL));
}
export async function onShutdown(server) {
  await server.dbPool.close();
}
```

After saving either file, `make dev-fastify` picks it up on next boot (no explicit registration).

## Parity Notes

Intentional drift with `mta-ployglot-server` Python twin:

- **File suffix:** `.mjs` here, `.py` there. The `routeAddon`/`lifecycleAddon` implementations use language-native discovery (`readdir` vs `pkgutil.iter_modules`) but same numeric-prefix rule.
- **Export shape:** Fastify routes export a default `async (fastify, config) => void` function; FastAPI routes export `mount(app, config) -> APIRouter`. Factory-style sync function on the Python side because `register_routes` (the adapter method) cannot await.
- **Lifecycle timing:** `onInit` runs synchronously during `setup()` on both sides. `onStartup` / `onShutdown` attach to `server.addHook('onReady'/'onClose')` here vs FastAPI's `lifespan` context manager.
- **Request state:** Fastify uses `server.decorateRequest("state", null)` + an `onRequest` hook to clone `initial_state` per-request. FastAPI uses ASGI middleware for the same effect.
- **Default port:** `51000` vs `52000` (host-side `FASTIFY_PORT=5100` vs `FASTAPI_PORT=5200` when docker-mapped).
