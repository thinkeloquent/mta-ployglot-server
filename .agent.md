# mta-ployglot-server — LLM Entry Reference

This repo is an **orchestration shell** for a polyglot meta-repo. It hydrates a registry of sibling repos into a local symlink topology, generates per-language workspace manifests (`pnpm-workspace.yaml`, `go.work`, virtual `Cargo.toml`), and assembles a release branch via `git subtree` — one squashed commit per sibling. It is **not** a deploy target.

It also ships **two runtime apps** under `server/fastify/` (mjs) and `server/fastapi/` (py) that exercise the addon-discovery contract from the bootstrap library at `ployglots/mta-ployglot-server-bootstrap/`. Each app has its own `.agent.md`; this file is the index plus the orchestration-shell entry point.

## SSOT

- [`workspace.toml`](./workspace.toml) is the single source of truth (every sibling repo + topology). See [`SSOT.md`](./SSOT.md).
- All derived projections are gated by `make projections-check`:
  - `.dev/workspace.toml.lock.json` — gitignored
  - `docker-compose.yml` — committed
  - `Makefile.entries` — committed
  - `pnpm-workspace.yaml` / `go.work` / virtual `Cargo.toml` — generated

## Workspace bootstrap

```bash
make bootstrap                               # clone missing siblings, lay symlinks, regenerate projections
make bootstrap ARGS=--dry-run                # preview
make bootstrap ARGS=--re-emit-only           # skip git fetch, just regenerate manifests
make bootstrap ARGS=--verify                 # report drift, exit non-zero on diff
make bootstrap ARGS=--only=pkg-vault         # narrow to one entry
```

Vault override: `WORKSPACE_OVERRIDE_<NAME_UPPER>=/abs/path` (vault-wins precedence over `local_path`).

The same logic is exposed as an AI-runnable skill at `.claude/skills/local-workspace-bootstrap/SKILL.md`.

## Multi-repo coordination

Sibling-spanning git operations. Every target has a `-json` twin and accepts `SIBLINGS="name1 name2"` for subset filtering (typos refuse loudly).

```bash
make status-all                  # cross-sibling state, exit 0 only if all clean + on default_ref
make sync-all APPLY=1            # pull default_ref everywhere
make branch-all NAME=feature/x   # coordinated branch creation
make commit-all MSG="…"          # commits on matching branches; refuses on protected refs
make push-all CONFIRM=1          # never exposes --force
```

## Quality gates (Checklist library)

`make pre-push` runs 8 gates in order, stop-on-fail:
`addon-lint → vault-check → twin-diff → projections-check → subtree-lint → security-scan → changelog-check → agent-md-check`.

```bash
make pre-push                                  # serial, stop-on-fail
make pre-push KEEP_GOING=1                     # CI mode, continue past failures
make pre-push-json | tee /tmp/gates.jsonl      # JSON-lines for dashboards
make checklist                                 # interactive walkthrough
VAULT_CHECK=warn make pre-push KEEP_GOING=1    # local opt-out when no .env* sourced
```

`make agent-md-rehash` refreshes the `<!-- surface-hash: … -->` markers embedded at the bottom of each per-app `.agent.md` after an intentional surface change; `make agent-md-check` verifies them.

`make security-scan` queries GitHub Advanced Security for secret-scanning alerts (no local gitleaks/trufflehog). Auth: `GH_TOKEN_SECSCAN` (preferred) or `GITHUB_TOKEN_SECSCAN`. `CI=1` hard-fails on missing tooling; dev shell warns and skips.

## Release assembly — PR-driven

The release branch is built by `git subtree` (copy-and-merge), not push-to-deploy. Cuts are manual; assembly happens in CI on PR merge; the manifest committed by the workflow is the durable record.

```bash
git checkout -b release/2026.05.0 origin/main && git push -u origin release/2026.05.0
gh pr create --base release/2026.05.0 --title "Release 2026.05.0 prep"   # triggers preflight workflow
gh pr merge --merge                                                       # triggers assemble workflow
git tag v2026.05.0 && git push origin v2026.05.0                          # tag root
make release-tag-propagate TAG=v2026.05.0 REF=release/2026.05.0 CONFIRM=1 # propagate to siblings
```

`make release-assemble` is reproduction-only and refuses without `MANIFEST=`. Workflows: `.github/workflows/release-preflight-on-pr.yml`, `.github/workflows/release-assemble-on-pr-merge.yml`.

## Composition contract — app-yaml chain

Lifecycle files are loaded in numeric-prefix order. The app-yaml chain claims slots 25..29 and writes the following onto `app.state` (fastify decorators have the same names, no leading `state.`):

| Slot | File                                              | Writes                                       | Reads                                   |
| ---- | ------------------------------------------------- | -------------------------------------------- | --------------------------------------- |
| 25   | `25_app_yaml_loader.lifecycle.{mjs,py}`           | `app_yaml_loader: LoaderHandle`              | (filesystem only)                       |
| 26   | `26_runtime_template_resolver.lifecycle.{mjs,py}` | `runtime_template_resolver: ResolverHandle`  | `env_resolve` (slot 15)                 |
| 27   | `27_app_yaml_config.lifecycle.{mjs,py}`           | `app_yaml_config: ConfigHandle / SDK`        | `app_yaml_loader` (slot 25)             |
| 28   | `28_app_yaml_from_context.lifecycle.{mjs,py}`     | `app_yaml_applier: ApplierHandle`            | `runtime_template_resolver` (slot 26)   |
| 29   | `29_app_yaml_fetch_config.lifecycle.{mjs,py}`     | `app_yaml_fetch_config: FetchConfigHandle`   | `app_yaml_config` + `app_yaml_applier`  |

Renaming or deleting any of these files breaks the chain. The numeric prefix is the contract.
