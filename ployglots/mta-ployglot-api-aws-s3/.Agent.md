# Agent Integration Guide

Tells downstream coding agents (Claude Code, Cursor, Aider, etc.) how to consume this repository.

## What this repo delivers

A polyglot package wrapping **the AWS S3 HTTPS client builder**:

- `packages/mjs/` — `@mta/polyglot-aws-s3`, a TypeScript Node ESM SDK exposing `SDKConfig`, `validateConfig`, `assertValidConfig`, and `buildS3Client`. Supports proxy URL, connect/read timeouts, max-retries, TLS verification (via `NODE_TLS_REJECT_UNAUTHORIZED`), path-style addressing, and custom endpoints.
- `packages/py/` — `polyglot-aws-s3`, a Python async-context-manager twin (`SDKConfig`, `validate_config`, `assert_valid_config`, `build_s3_client`) built on `aiobotocore` + `botocore.Config`.

There is no CLI, no env-resolver, and no standalone HTTP service. The "API" surface is the public function contract of both packages — env-resolution and verification helpers are planned but belong to an app-integration plan (Features 01 and 03 in the tracked plan are out of scope for this repo).

## Integration instruction

When working in this repo, the agent SHOULD:

1. **Read `README.md`** at the repo root for the high-level layout.
2. **Read the active plan** at `../AI-Agent-Plans/aws-s3-polyglot-plan-20260419-b7f3e2a1/` if making structural changes — features, stories, and tasks live there.
3. **Walk `examples/`** as the executable contract — every example must run cleanly against the implemented code before any change is considered done.
4. **Run the test suite** before any change is reported complete:
   ```
   pnpm install            # at repo root (workspace install)
   pnpm test               # delegates to packages/mjs vitest
   cd packages/py && make install && make test
   ```
   All tests must pass.

## Usage

### Verify the library

```
pnpm install
pnpm --filter "./packages/mjs" run test

cd packages/py && make install && make test
```

Expected: `# pass N` across `packages/mjs/test/**/*.test.ts` and green pytest run for `packages/py/test/`.

### Use the Node SDK

```ts
import { buildS3Client, type SDKConfig } from "@mta/polyglot-aws-s3";

const config: SDKConfig = {
  bucketName: "my-bucket",
  region: "us-east-1",
  forcePathStyle: false,
  connectTimeout: 10,
  readTimeout: 60,
  maxRetries: 3,
  verifySsl: false,
  // proxyUrl: process.env.HTTPS_PROXY,          // optional
  // endpointUrl: "http://localstack:4566",     // optional
};

const handle = await buildS3Client(config);
try {
  // handle.client is a fully configured @aws-sdk/client-s3 S3Client
} finally {
  await handle.destroy();
}
```

See `examples/sdk/` for runnable scenarios.

### Use the Python SDK

```py
import asyncio
from polyglot_aws_s3 import SDKConfig, build_s3_client

async def main():
    cfg = SDKConfig(bucket_name="my-bucket", region="us-east-1")
    async with build_s3_client(cfg) as client:
        r = await client.list_buckets()
        print([b["Name"] for b in r.get("Buckets", [])])

asyncio.run(main())
```

## Conventions

- Source paths in plan tasks are absolute or rooted at the repo top-level.
- Examples are runnable as-is given the relevant env var; secrets never hardcoded.
- Deviations from the plan are recorded in the plan's task or feature file under a `## Deviations` section.

## Notable plan deviations (in this implementation)

- Only Feature 02 (the S3 client builder + `SDKConfig` surface) is implemented here. Feature 01 (env-resolver `configFromEnv` / `config_from_env`) and Feature 03 (`verifyConnectivity` / `verify_connectivity`) are out of scope for this package and belong to the app-integration plan.
- `packages/mjs/` uses TypeScript + pnpm + vitest (the plan prescribes this toolchain explicitly in Feature 02 tasks), not the bare `.mjs` + `node --test` pattern used by other polyglot repos under this account.
- Python package is distributed as `polyglot-aws-s3` (module `polyglot_aws_s3`), matching the `polyglot-<slug>` naming used by sibling repos.
