```

examples/fastify-app → runnable smoke app          (not published)
```
Install from repo root: `npm install` (workspace-aware).
Deterministic CI install: `npm ci` at repo root.

**Python side** — `pyproject.toml` `[tool.uv.workspace]` members:

## Cross-cutting conventions

- **Same purpose, different framework.** Both packages print a route table to stdout. Neither affects routing or request handling.
- **Extract-only.** Both packages are extracted verbatim from the upstream monorepo (`/Users/Shared/autoload/A08b2d3148c2a49f49d710a5f6b36c8e1/platform/`) with no behaviour changes.
- **Zero platform-core assumptions.** Neither package imports anything from the host monorepo. They work with vanilla Fastify / FastAPI instances.
- **Makefile is authoritative.** `make ci` is the single CI command for each package. Package-manager scripts (`npm test`, `pytest`) are invoked by the Makefile — use `make <target>` directly.
- **Same Makefile shape.** Both packages expose: `install`, `ci-install`, `test`, `test-watch`, `lint`, `build`, `clean`, `distclean`, `ci`. Targets are equivalent in intent across runtimes.
- **MIT licence** — both packages.
- **Methods sorted alphabetically** — both packages sort HTTP methods (GET, POST, …) alphabetically within each route line.

## Intentional drift

These are deliberate differences between the twins. Do **not** align them.

| Dimension | Node/MJS (`packages/mjs/`) | Python (`packages/py/`) |
|-----------|---------------------------|------------------------|
| **API shape** | Two-step: `setupRouteCollector(app)` (before routes) + `printRoutes(app)` (after ready) | One-step: `print_routes(app)` (after routes registered) |
| **Collection mechanism** | `onRoute` hook fires at registration time; state stored in `app._routeMap` (`Map<url, Set<method>>`) | Reads `app.routes` directly at call time; no stored state |
| **Fallback** | `printRoutes` falls back to `fastify.printRoutes({ commonPrefix: false })` when `_routeMap` is absent or empty | No fallback; silently skips non-`APIRoute` entries |
| **Output columns** | `<url padEnd 55> \| <methods>` | `<path:<40> \| <methods:<10> \| <name>` — Python adds route name |
| **Exports** | Named: `setupRouteCollector`, `printRoutes`; two subpath exports (`.` and `./print-routes`) | Named: `print_routes`; `__all__ = ["print_routes"]` |
| **Module format** | ESM-only (`.mjs` extension, `"type":"module"`); no CJS build | N/A (Python has no module-format restriction) |
| **Runtime version** | Node ≥ 20 | Python 3.11–3.12 (`requires-python = ">=3.11,<3.13"`) |
| **Build artefact** | Tarball via `npm pack` (pure ESM, no transpile; `make build` is a no-op) | sdist + wheel via `hatchling` |
| **Test runner** | `node --test` | `pytest` |
| **Lint tool** | `node --check` (syntax only) | `ruff` |
| **URL column padding** | 55 chars | 40 chars |
