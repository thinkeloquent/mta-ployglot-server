# Agent Integration Guide — mta-ployglot-pkg-app-yaml-loader

## What this package delivers

A polyglot library (Node ESM `.mjs` + Python 3.11+) that loads an ordered list of YAML files and returns the parsed contents as a path-keyed map. No merge, no resolution, no validation — pure file IO.

Two siblings consume its output:

- `app_yaml_config` — merge + provider semantics on top of the map.
- `app_yaml_overwrite` — `{{env.VAR}}` / `{{fn:NAME}}` template resolution on parsed values.

## Public surface

| Symbol (mjs)        | Symbol (py)            | What it does |
| ------------------- | ---------------------- | ------------ |
| `loadFiles`         | `load_files`           | Read + parse + return ordered map keyed by absolute path. |
| `loadFromConfigDir` | `load_from_config_dir` | Compose `resolve*` + `buildConfigFiles` + `loadFiles`. |
| `resolveConfigDir`  | `resolve_config_dir`   | override → `CONFIG_DIR` → `<callerDir>/../../../../common/config` → throw. |
| `resolveAppEnv`     | `resolve_app_env`      | `(override \|\| APP_ENV \|\| "dev").toLowerCase()`. |
| `buildConfigFiles`  | `build_config_files`   | Build canonical 6-file list (4 base + 2 env-suffixed). |
| `clearCache`        | `clear_cache`          | Evict one or all entries from the in-memory cache. |
| `LoadError`         | `LoadError`            | Wraps read/parse failures with `.path` + `.cause`. |

## Conventions

- Iteration order = insertion order = file priority. Callers downstream rely on this.
- Empty files (`---` or zero-byte) yield `{}`, not `null` (deep-merge-safe).
- Cache is **process-local**; cross-process invalidation is out of scope.
- mjs ↔ py kept in lockstep — public symbols match concept-for-concept; only naming case differs.
