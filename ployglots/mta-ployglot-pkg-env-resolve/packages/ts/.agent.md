# env-resolve (ts)

Four-tier configuration resolver (`arg → env → config → default`) with typed coercion wrappers (`bool`, `int`, `float`). TypeScript twin of `packages/py/`. Behavior is canonical per the repo-root [`SPEC.md`](../../SPEC.md).

## Install

```bash
npm i @org/env-resolve
```

Zero runtime dependencies. ESM-only. Requires Node ≥ 20.

## Import Surface

Re-exported from `src/index.ts`:

- `resolve(arg, envKeys, config, configKey, defaultValue)` — generic four-tier resolver.
- `resolveBool(arg, envKeys, config, configKey, defaultValue)` — coerces to `boolean`.
- `resolveInt(arg, envKeys, config, configKey, defaultValue)` — coerces to integer; rejects decimal-shaped strings (D4) and `boolean` inputs (D5).
- `resolveFloat(arg, envKeys, config, configKey, defaultValue)` — coerces to float; rejects partial-numeric strings (D6).
- `TRUTHY_STRINGS` — readonly tuple `['true', '1', 'yes', 'on']`. Case-insensitive members are truthy under `resolveBool`.

## Canonical Usage

```ts
import { resolve, resolveBool, resolveInt } from '@org/env-resolve';

// Tier 1 — explicit arg always wins.
const region = resolve(
  cliFlags.region,                // arg
  ['REGION', 'AWS_REGION'],       // envKeys (first-match wins)
  appConfig,                      // config
  'region',                       // configKey
  'us-east-1',                    // defaultValue
);

// Coercion cascade: env-driven feature flag.
const enabled = resolveBool(undefined, 'FEATURE_FLAG', null, null, false);

// Strict integer parsing — "3.14" returns the default, not 3.
const port = resolveInt(undefined, 'PORT', null, null, 8080);
```

## Integration

- **Depends on (monorepo):** none.
- **Depends on (external):** none at runtime. Dev-only: `typescript`, `tsx`, `@types/node`.
- **Environment:** reads any key passed via `envKeys` through `process.env`. No fixed env-var names.

## Parity Notes

- Surface is naming-convention difference only: `resolveBool` ↔ `resolve_bool`, `envKeys` ↔ `env_keys`, etc. Behavior is identical by spec.
- TS reads env via `process.env[k]`; Python via `os.getenv(k)`. Both return `string | undefined` / `str | None`, so callers see the same shape.
- D5 (bool rejected by `resolveInt`) requires a runtime guard in both ports: `typeof === 'boolean'` (TS) and `isinstance(val, bool)` *before* the int arm (py, since `bool` subclasses `int`).
- See [`/.agents/parity.md`](../../.agents/parity.md) for the full drift catalog.
