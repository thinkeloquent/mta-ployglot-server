# Agent Integration Guide — mta-ployglot-pkg-app-yaml-config

A polyglot library (mjs + py) exposing a deep-merged, read-only configuration store on top of an ordered map of parsed YAML files. Consumes `app-yaml-loader`'s output; consumed by `app-yaml-overwrite` and `app-yaml-fetch-config`.

## Packages

| Package | Path | Entry |
| ------- | ---- | ----- |
| `@ployglot/app-yaml-config` (Node ESM) | `packages/mjs/` | `src/index.mjs` |
| `app_yaml_config` (Python ≥ 3.11) | `packages/py/` | `src/app_yaml_config/__init__.py` |

## Public surface

- `AppYamlConfig` — singleton class. `initialize / getInstance / _resetForTesting / restore`, plus readers (`get`, `getNested`, `getAll`, `getGlobalAppConfig`, `getOriginal`, `getOriginalAll`) and immutable mutator stubs (`set`, `update`, `reset`, `clear` → `ImmutabilityError`).
- `AppYamlConfigSDK` — thin façade. `fromDirectory(configDir)` async factory; `listProviders / listServices / listStorages`; dot-path `get(path, default)`.
- Pure functions — `deepMerge`, `mergeFiles`, `mergeGlobalIntoProviders` (no class state).

## Conventions

- Singleton state is process-local. Tests must call `AppYamlConfig._resetForTesting()` between cases.
- Returned objects are deep-cloned. Mutation never poisons the store.
- `initialize` accepts either `loaded: Map<path, parsed>` (canonical) or `files`/`configDir` (compat — requires the optional `app-yaml-loader` dep).

## CI

Each package ships an eight-target Makefile (`help install ci-install lint test build clean distclean ci`).

```
cd packages/mjs && make ci
cd packages/py  && make ci
```

## Examples

`examples/sdk/` carries embedding scenarios; `examples/api/` documents the public surface entries. Both assume `make build` has been run in the package you're calling from.
