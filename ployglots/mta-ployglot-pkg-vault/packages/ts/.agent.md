# vault-file (ts)

`@polyglot/vault-file` — vault file + env-store SDK for Node.js. TypeScript
twin of `polyglot-vault-file` (py).

## Install

```bash
npm i @polyglot/vault-file
```

Engines: Node ≥ 20. ESM only (`"type": "module"`).

## Import Surface

Classes:
- `EnvStore` — process-wide singleton (`getInstance`, `onStartup`, `onStartupAsync`, `get`, `getOrThrow`, `isInitialized`)
- `VaultFileSDK`, `VaultFileSDKBuilder` — fluent builder + 11-method SDK
- `Logger`, `EnvKeyNotFoundError` (with `.key` attribute)

Functions:
- `fromJSON`, `toJSON`, `normalizeVersion`, `parseEnvFile` (from `core.ts`)
- `getLogger`, `setLogLevel`, `makeSDKError`

Types / interfaces / schemas:
- `VaultFile`, `VaultHeader`, `LoadResult`
- `VaultFileSchema`, `VaultHeaderSchema` (zod)
- `SDKResult<T>`, `SDKError`, `SDKInfo`, `ConfigDescription`, `SecretInfo`, `ValidationResult`, `DiagnosticResult`
- `IVaultFile`, `IEnvStore`, `IVaultFileLogger`, `IVaultFileSDK`
- `LogLevel` (enum)

## Canonical Usage

```ts
import { VaultFileSDK } from '@polyglot/vault-file';

const sdk = VaultFileSDK.create().withEnvPath('.env').build();
const result = sdk.loadConfig();
if (!result.success) {
  console.error('load failed:', result.error);
  process.exit(1);
}
console.log(JSON.stringify(result.data));  // { "totalVarsLoaded": 45 }
```

## Integration

- **Depends on (monorepo):** none (workspace member under `packages/ts`).
- **Depends on (external):** `zod` (runtime).

## Parity Notes

See [`/.agents/parity.md`](../../.agents/parity.md) for the full twin-drift table. Highlights:

- **`EnvStore.get` priority is vault-wins** on both twins: `store → process.env → default`. Breaking change vs pre-1.0.0 TS source (which had `process.env → store`).
- **`EnvStore.onStartup` is synchronous** on both twins. TS additionally exports `onStartupAsync` returning `Promise<LoadResult>` for callers that want an awaitable.
- **`EnvKeyNotFoundError.key`** is exposed on the instance on both twins; message is exactly `"Environment variable '<key>' not found"`.
- **`SDKError`** is a named interface on TS (plus `makeSDKError` helper); on py it is a `BaseModel` class.
- **`VaultFileSDKBuilder.withLogger`** is present on both; injected logger falls back to a module default when omitted.
