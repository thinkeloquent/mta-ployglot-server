# vault-file (py)

`polyglot-vault-file` — vault file + env-store SDK for Python. Python twin
of `@polyglot/vault-file` (ts).

## Install

```bash
pip install polyglot-vault-file
# or, inside this uv workspace:
uv sync
```

Python ≥ 3.11, < 3.13. Runtime dep: `pydantic ≥ 2.12.5`.

## Canonical Usage

```python
from polyglot_vault_file import VaultFileSDK

sdk = VaultFileSDK.create().with_env_path(".env").build()
result = sdk.load_config()
if not result.success:
    print(f"load failed: {result.error}")
    raise SystemExit(1)
print(result.data.model_dump_json(by_alias=True))
# {"totalVarsLoaded":45}
```

## Parity Notes

See [`/.agents/parity.md`](../../.agents/parity.md) for the full twin-drift table. Highlights:

- **`EnvStore.get` priority is vault-wins** on both twins: `store → os.environ → default`. The py twin's behaviour is the canonical one that the TS twin now mirrors.
- **`EnvStore.on_startup` is synchronous** on both twins. Py does not need an async adapter — `on_startup` already returns `LoadResult` directly.
- **`LoadResult` wire format:** attribute access is snake_case (`.total_vars_loaded`); JSON serialization via `model_dump_json(by_alias=True)` emits camelCase (`"totalVarsLoaded"`) to match the TS twin.
- **`VaultHeader.created_at`** default is millisecond precision ISO 8601 (`datetime.isoformat(timespec="milliseconds")`); TS emits the same precision via `Date#toISOString`. The two formats differ only in trailing `+00:00` (py) vs `Z` (ts).
- **`VaultFileSDKBuilder.with_logger`** was the origin of this capability; the TS twin added a matching `withLogger` in 1.0.0.
