# `@polyglot/fetch-http-client` — Node/TypeScript skill

Async HTTP client for Node ≥ 18 built on [`undici`](https://github.com/nodejs/undici). ESM-only. API-compatible with `fetch-httpx` (the Python reference whose surface this package mirrors). Python twin lives at [`../py/`](../py/).

## Install

```bash
npm install @polyglot/fetch-http-client undici
```

`undici` is a **peer dependency** (`^7.19.0`) — declared `optional: true` so the package installs without it, but every real workload pulls it in. Deduping across monorepos is the motive.

No other runtime dependencies. Node `>=18.0.0`. Type: `"module"` — ESM only, no CJS build.

## Canonical usage pattern

```ts
import { AsyncClient, BasicAuth } from '@polyglot/fetch-http-client';

const client = new AsyncClient({
  baseUrl: 'https://api.example.com',
  auth: new BasicAuth('user', 'pass'),
  timeout: 5000,
  retry: { maxRetries: 3, retryOnStatus: [429, 502, 503, 504] },
});

try {
  const resp = await client.get('/users/me');
  resp.raiseForStatus();
  const me = await resp.json<{ id: string; name: string }>();
} finally {
  await client.close();          // always close — undici pools are long-lived
}
```

**Always `await client.close()`** in a `finally` block. The client holds undici pools which will otherwise leak handles and keep the event loop alive.

## Proxy convention

Every example accepts `proxy = {}` — the empty bag means "auto-detect from `HTTPS_PROXY` / `HTTP_PROXY`":

```ts
import { buildProxy } from '../examples/_shared.js';

const proxy = buildProxy({});             // auto-detect
const explicit = buildProxy({
  host: 'http://corp-proxy.internal:3128',
  user: 'svc',
  pass: process.env.PROXY_SECRET,
});

const client = new AsyncClient({
  baseUrl: 'https://api.example.com',
  ...(proxy ? { proxy } : {}),            // omit entirely when undefined
});
```

`NO_PROXY` is honored. When `buildProxy(...)` returns `undefined`, callers **omit** the `proxy:` field — do not pass `proxy: undefined`.

## Retry + resilience defaults

- Retry fires only for methods in `IDEMPOTENT_METHODS` (`GET`/`HEAD`/`OPTIONS`/`TRACE`/`PUT`/`DELETE`). `POST` / `PATCH` never retry — don't "fix" this.
- `JitterStrategy`: `NONE` | `FULL` | `EQUAL` | `DECORRELATED`. `FULL` is the default.
- `Retry-After` header is honored for `429` / `503`.
- Circuit breaker states: `CLOSED` → `OPEN` → `HALF_OPEN`. `CircuitOpenError` short-circuits while open.
- `RETRYABLE_ERROR_CODES` + `isRetryableError()` classify undici transport errors.

## Cache contract

Only `GET` / `HEAD` responses with status in `{200, 203, 300, 301, 404, 410}` are cached (RFC 7234-ish). Wrap a client:

```ts
import { AsyncClient, CachingClient } from '@polyglot/fetch-http-client';

const inner = new AsyncClient({ baseUrl: 'https://api.example.com' });
const client = new CachingClient(inner, { ttlSeconds: 60 });
```

Or apply cache-aware middleware via `createCacheAwareClient` / `createCacheHooks`.

## Logger env contract

Driven by environment variables:

| Var          | Values                                           | Default                    |
| ------------ | ------------------------------------------------ | -------------------------- |
| `LOG_LEVEL`  | `TRACE` / `DEBUG` / `INFO` / `WARN` / `ERROR` / `SILENT` | `INFO`              |
| `LOG_FORMAT` | `json` / `pretty`                                | `pretty` in dev, `json` in prod |
| `NODE_ENV`   | `development` / `production`                     | (defaults pretty → json)   |

Secrets in headers are **auto-redacted** (`authorization`, `cookie`, `set-cookie`, `x-api-key`, common bearer patterns). Don't disable unless you know why.

## Dev loop — Makefile is source of truth

Run everything via `make` from `packages/ts/`. The Makefile is authoritative; npm scripts in `package.json` are thin wrappers.

```
make help              # list all targets
make ci                # ci-install → lint → test → build (full CI pipeline)
make install           # npm install at workspace root
make ci-install        # npm ci at workspace root (frozen lockfile)
make lint              # prettier --check src  +  tsc --noEmit
make lint-fix          # prettier --write src
make typecheck         # tsc --noEmit only
make test              # vitest run
make test-watch        # vitest watch mode
make test-integration  # live-provider tests (env-gated)
make coverage          # vitest with coverage report
make build             # tsup → dist/
make pack              # npm pack → .tgz
make clean             # remove dist/, *.tgz, coverage/
make distclean         # clean + remove node_modules
```

Note: `make install` runs `npm install` from `$(PKG_DIR)/../..` — the workspace root — not inside `packages/ts/`. Don't run `npm install` inside the package directly.

## Parity with the Python twin

When adding a feature here, check [`../py/.agent.md`](../py/.agent.md) before you finish. Enforced parity surface:

- Same auth class names (`BasicAuth`, `BearerAuth`, `APIKeyAuth`, `DigestAuth`).
- Same `Timeout` phases (connect/read/write/pool) and `DEFAULT_LLM_TIMEOUT`.
- Same retry + jitter strategies (`NONE`/`FULL`/`EQUAL`/`DECORRELATED`).
- Same circuit-breaker states (`CLOSED`/`OPEN`/`HALF_OPEN`).
- Same 6 integration examples, same env contract, same `proxy={}` placeholder.

Intentional drift:

- TS = `camelCase`, Python = `snake_case`.
- TS has streaming now; Python defers it.
- TS has a richer SDK facade (`@polyglot/fetch-http-client/sdk`); Python has only `fetch_httpx_async`.
- Framework adapters (Fastify here, FastAPI on the Python side) are both deferred — plan feature 13.

## Gotchas

- **ESM only** — no `require()`. Import maps are defined in `package.json` `exports`; the `types` condition must come before `import` only if you're patching the exports map (current form is correct, types emit alongside JS).
- **Undici version lock** — package + peer are both pinned to `^7.19.0`. Do not diverge them.
- **Close leaks** — `AsyncClient` / `AsyncClientPool` hold undici pools; failure to `close()` keeps the event loop alive and leaks sockets. `closeDefaultClient()` exists for the module-level verbs.
- **Workspace install** — always install from the workspace root, not inside `packages/ts/`. The Makefile enforces this; `npm install` run inside the package will desync the lockfile.
