# Agent Integration Guide

Tells downstream coding agents (Claude Code, Cursor, Aider, etc.) how to consume this repository.

## What this repo delivers

A polyglot package wrapping **GitHub Projects v2**:

- `packages/mjs/` — `@mta/github-projects`, a functional Node ESM SDK with full CRUD coverage across Projects, Fields, Items, Field Values, Views, Access, Relationships, Attachments, and Bulk operations. Supports custom HOST (Enterprise Server) and optional HTTP/HTTPS proxy.
- `packages/py/` — `polyglot-github-projects`, a Python placeholder package. Every public function raises `NotImplementedError` and points at the Node sibling. The package locks the public surface for a future port.

There is no CLI and no standalone HTTP service. The "API" surface is the public function contract of `@mta/github-projects`.

## Integration instruction

When working in this repo, the agent SHOULD:

1. **Read `README.md`** at the repo root for the high-level layout.
2. **Read the active plan** at `../AI-Agent-Plans/gh-projects-polyglot-api-20260418-a7f3c2e1/` if making structural changes — features, stories, and tasks live there.
3. **Walk `examples/`** as the executable contract — every example must run cleanly against the implemented code before any change is considered done.
4. **Run the Node test suite** before any change is reported complete:
   ```
   npm test
   ```
   The full suite is 187 tests; all must pass.

## Usage

### Verify the library

```
npm install
npm test
```

Expected: `# pass 187` across `packages/mjs/test/**/*.test.mjs`.

### Use the Node SDK

```js
import { createClient } from '@mta/github-projects';

const client = createClient({
  token: process.env.GITHUB_TOKEN,
  host: process.env.GITHUB_HOST,    // optional — defaults to api.github.com
  proxy: process.env.HTTPS_PROXY,   // optional
});

const project = await client.projects.get({ owner: 'acme', number: 42 });
```

See `examples/sdk/` for runnable scenarios.

### Verify the Python stub

```
python3 -c "import polyglot_github_projects as p; print(p.__version__)"
```

Expected: `0.1.0`. Every public function raises `NotImplementedError`.

## Conventions

- Source paths in plan tasks are absolute or rooted at the repo top-level.
- Examples are runnable as-is given a `GITHUB_TOKEN` env var; secrets never hardcoded.
- Deviations from the plan are recorded in the plan's task or feature file under a `## Deviations` section.

## Notable plan deviations (in this implementation)

- Test script uses a glob (`'test/**/*.test.mjs'`) rather than a directory arg — Node 22 changed `--test` directory semantics.
- Proxy integration test verifies the dispatcher is forwarded to `fetch` rather than performing a real round-trip through a stub proxy (undici 6 + Node 22 made the round-trip flaky).
- `SET_VALUE` mutation returns a slim `{id type isArchived}` shape instead of the full `ItemFields` fragment to break a circular import between `graphql/values.mjs` and `graphql/items.mjs`.
- `client._internals.rawPost` was added to support the absolute-URL upload endpoint that the attachments upload task assumed.
