# SSOT — Single Source of Truth

**SSOT** stands for **Single Source of Truth**.

It's the principle that for any given piece of data, exactly one place owns the canonical value, and every other representation is *derived* from it (and must be regeneratable + verifiable).

## In this repo

- **SSOT:** [`workspace.toml`](./workspace.toml) — the hand-edited registry of sibling repos.
- **Derived projections** (regenerated from the SSOT, never hand-edited):
  - [`docker-compose.yml`](./docker-compose.yml) — **committed**; registry-driven `build.additional_contexts` per service. Hand-editing forbidden; CI gates staleness.
  - `.dev/workspace.toml.lock.json` — **local only** (`.dev/` is gitignored); JSON projection for non-TOML consumers (JS scripts, agents, GitHub Actions). Regenerated by `make init` / `make bootstrap`.
  - `Makefile.entries`, `pnpm-workspace.yaml`, `go.work`, `Cargo.toml` — per-language manifests emitted by `scripts/workspace/emit-*.sh`.

## How the invariant is enforced

The `--check` modes on each emitter, plus the CI gate in [`.github/workflows/release-assemble.yml`](./.github/workflows/release-assemble.yml), exist precisely to enforce the SSOT invariant: **if anyone edits a derived file by hand, or edits the SSOT without re-emitting, CI fails.**

| Concern                         | Command                          |
|---------------------------------|----------------------------------|
| Regenerate every projection     | `make projections`               |
| Verify nothing is stale (CI)    | `make projections-check`         |
| Regenerate just the lock JSON   | `make workspace-lock-emit`       |
| Regenerate just docker-compose  | `make docker-compose-emit`       |

The release-assemble workflow runs `make projections-check` before subtree assembly, so a release branch with a stale projection cannot be assembled.

## Why a TOML SSOT plus JSON projection

`workspace.toml` is the SSOT because it's small, comment-friendly, and validated against `scripts/workspace/schema/workspace.schema.json`. Some consumers (Node scripts, agents, GitHub Actions step expressions) can't read TOML without taking on a parser dependency, so `.dev/workspace.toml.lock.json` provides a stable, schema-versioned JSON shape with a `source_sha256` field pointing back at the TOML — any consumer can verify it's reading a projection of the current SSOT. The JSON lives under `.dev/` (gitignored) since it's a derivable, local artifact rather than a source-tree contract.
